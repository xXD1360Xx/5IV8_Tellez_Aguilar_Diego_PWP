Todos los formulacons en la web manejan métodos para el evnío de las diferentes peticiones que reliza el usuario. Considerando la arquitectura cliente servidor, el manejo de las petiuciones se hac epor medio todos los formularios en la web, manejan respues

    GET:
      Este metodo se encarga de enviar la información del formulario por medio de la salida estandar (conocida como URL) del navegador, esto propicia que puedan existir inyecciones de codigo realizando violaciones a la seguridad del sistema.

    POST:
      Este metodo se encarga de enviar la informaicón para poder registrar un cambio o un nuevo registro dentro del sistema, enviandola igual por el metodo estandar, pero mediante un enrutamiento, esto significa que no esvisible para el usuario, los que nos ofreece un mayor grado de seguridad en el manejo de información.


En resumen, los métodos son:

    • GET: Sirve para enviar y realizar operaciones de lectura o consultas
      POST: Sirve para enviar información de un recurso al servidor, con el fin de registrarlo
      Delete: Sirve para eliminar un registro especifico dentro del servidor, obteniendo una respuesta de actualización
      Put: Sirve para almacenar la información de un recurso particular, y conectarse al servicio para realizar la actualización
      Head: Sirve para relizar el código de respuesta de cualquier cabecera http, asociada a algún recurso
